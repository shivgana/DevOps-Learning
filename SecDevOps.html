DevOps Introduction

DevOps is a set of practices that combines software development and IT operations. It aims to shorten the systems development life cycle and provide continuous delivery with high software quality.

  • Divides an application into microservices for rapid development while maintaining quality and integrity.
  • DevOps is the methodology of Agile, Continuous Integration, Continuous delivery and Continuous deployment.

Secure DevOps

DevOps security is a philosophy that combines three words: development, operations, and security. The goal is to remove any barriers that may exist between software development and IT operations.

Integration of Security in DevOps

Security check for Code Owner (PAM)
Security check for Code Build (SAST, Bugs, Vulnerability)
Security check for Application (DAST)

Tools for Security check

  • SAST - Sonarqube
  • Vulnerability - “OWASP dependency check” / Xray / Trivy
  • DAST - “OWASP ZAP”

SevDevOps for Jenkins, Git, Maven

  • Initially, the term “security” was included and explained in the DevOps concept, but it was not preferred.
